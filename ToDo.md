## 项目任务拆分

按照**由简到难、能跑能用**的原则拆分，每个阶段结束都有可运行的成果。

------

## 第一阶段：项目初始化 & 基础搭建

**目标：把项目骨架跑起来**

```
后端
  □ 创建 Spring Boot 3.x 项目（Spring Initializr）
  □ 集成 MyBatis-Plus + MySQL 配置
  □ 集成 Redis 配置
  □ 集成 Sa-Token 配置
  □ 集成 Knife4j 接口文档
  □ 统一返回格式封装 R<T>
  □ 全局异常处理
  □ 跨域配置

前端
  □ 创建 Vue 3 + TypeScript + Vite 项目
  □ 集成 Element Plus
  □ 集成 Pinia 状态管理
  □ 集成 Vue Router 4
  □ 封装 Axios 请求（统一拦截器、错误处理）
  □ 配置开发环境代理

部署基础
  □ 编写 docker-compose.yml（MySQL + Redis）
  □ 本地 Docker 环境跑通
```

**验收标准：** 前后端项目能启动，Knife4j 文档能访问，数据库连接正常。

## 第二阶段：用户系统 & 鉴权

**目标：完整的登录注册流程**

```
数据库
  □ 设计 user 表

后端
  □ 注册接口（用户名/密码）
  □ 登录接口（返回 Token）
  □ 退出登录接口
  □ Sa-Token 路由拦截配置
  □ 获取当前登录用户信息接口
  □ 密码加密（BCrypt）
  □ Token 存入 Redis

前端
  □ 登录页面
  □ 注册页面
  □ Axios 请求头自动携带 Token
  □ 路由守卫（未登录跳转登录页）
  □ 退出登录功能
  □ 基础布局框架（侧边栏 + 顶部导航 + 内容区）
```

**验收标准：** 能注册、能登录、未登录访问页面会跳转、登录后能看到主界面。

## 第三阶段：记账模块

**目标：第一个核心业务功能上线**

```
数据库
  □ 设计 account_bill 表（收支记录）
  □ 设计 bill_category 表（分类：餐饮/交通/娱乐...）

后端
  □ 账单 CRUD 接口
  □ 按月/按分类查询接口
  □ 收支统计接口（本月总收入、总支出、结余）

前端
  □ 记账列表页
  □ 新增/编辑账单表单
  □ 删除账单
  □ 按月份筛选
  □ 收支统计展示（用 Element Plus 图表或 ECharts）
```

**验收标准：** 能增删改查账单，能看到本月收支统计。

## 第七阶段：每日热点 & 金句

**目标：接入第三方数据**

```
后端
  □ 接入每日新闻 API（推荐：天行数据 / 聚合数据）
  □ 接入每日金句 API
  □ Redis 缓存每日数据（避免频繁请求）
  □ 定时任务每日自动刷新（Spring @Scheduled）

前端
  □ 首页展示每日热点新闻列表
  □ 首页展示每日金句
  □ 手动刷新按钮
```

**验收标准：** 首页能看到当日新闻和金句，数据有缓存不会重复请求。
